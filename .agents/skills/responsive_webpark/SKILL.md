# Webpark Responsive Guidelines

ทักษะนี้มีไว้เพื่อให้ AI ปฏิบัติตามกฎและข้อบังคับในการทำงานโปรเจค **Corporate Webpark** อย่างเคร่งครัด

## หน้าที่และความรับผิดชอบ (Scope of Work)
1. **งานที่ได้รับมอบหมาย**: ทำ Responsive ให้กับหน้า **บทความ (Article)** และ **ติดต่อเรา (Contact Us)** เท่านั้น
2. **ห้ามแก้ไขนอกเหนือจากที่ได้รับมอบหมาย**: โปรเจคนี้เป็นโปรเจคที่รับช่วงต่อมา ห้ามแตะต้องหรือแก้ไขไฟล์ หน้า หรือส่วนที่ไม่เกี่ยวข้องโดยเด็ดขาด 
3. **การแก้ไขโค้ด**:
   - พยายามแก้ไขโครงสร้างหลัก (HTML Structure / Core CSS) ให้น้อยที่สุด
   - ใช้วิธีเพิ่มคลาสหรือเขียน CSS เฉพาะจุด (Tailwind หรือ Custom CSS ที่มีอยู่แล้ว) เพื่อปรับหน้าจอสำหรับ Mobile/Tablet
4. **ความถูกต้องระดับ Pixel-Perfect**: 
   - ต้องทำหน้าตา Responsive ให้ตรงกับ UI ใน Figma แบบเป๊ะๆ (Pixel Perfect) 
   - ห้ามคิดหรือดัดแปลง UI เองเด็ดขาด หากติดปัญหาให้ยึด Figma เป็นหลัก

## โครงสร้างโปรเจคที่เกี่ยวข้อง
- **หน้าเว็บ (Frontend)**: โค้ดสำหรับแสดงผลอยู่ภายในโฟลเดอร์ `frontend/app/views/`
  - ส่วนของหน้าต่างๆ จะอยู่ใน `frontend/app/views/pages/`
  - `routes.php` (ภายใน `frontend/`) ทำหน้าที่ควบคุม URL เช่น `/article` และ `/contact`
- **ระบบหลังบ้าน (Admin)**: อยู่ที่โฟลเดอร์ `admin/` (ห้ามยุ่งเกี่ยว เพราะไม่ได้อยู่ใน Scope)

## คำแนะนำในการทำงาน
- ตรวจสอบไฟล์ใน `frontend/app/views/pages/article.php` (หรือที่เกี่ยวข้อง) และ `contact.php` 
- ก่อนแก้ ให้ดูโครงสร้างเดิมเสมอ และพิจารณาให้รอบคอบก่อนลบหรือเปลี่ยน Tag สำคัญ
